# 📋 리액트 프로그래밍 학습 노트 (03)

JSX의 개념부터 가상 DOM의 동작 원리까지, 리액트의 핵심 렌더링 메커니즘을 정리합니다.

---

## 1. JSX (JavaScript XML) 이해하기

### ✅ 정의 및 특징
* **정의:** 자바스크립트 코드 안에서 HTML과 유사한 마크업을 작성할 수 있도록 확장된 문법입니다.
* **변환 과정:** 웹 브라우저는 JSX를 직접 읽지 못하므로, **Babel(바벨)**이라는 도구가 실행 전에 일반 자바스크립트 코드로 변환해줍니다.
* **장점:** UI 구조를 직관적으로 파악할 수 있어 가독성이 뛰어나며, 로직과 디자인을 한 곳에서 관리할 수 있습니다.

---

## 2. 가상 DOM (Virtual DOM)의 마법

리액트가 실제 브라우저 화면을 업데이트하기 전, 메모리상에 먼저 그려보는 '설계도' 단계입니다.

* **동작 원리:** 1. 데이터가 변경되면 리액트는 전체 UI를 **가상 DOM**에 먼저 그립니다.
  2. 이전 가상 DOM과 현재 가상 DOM을 비교하여 **딱 바뀐 부분만** 찾아냅니다.
  3. 바뀐 조각들만 실제 브라우저 DOM에 적용(Patch)하여 불필요한 연산을 줄이고 속도를 높입니다.

---

## 3. 실습 코드 리뷰: 2진수 시계 (Binary Clock)

매 순간 데이터가 변할 때 리액트가 어떻게 효율적으로 대처하는지 확인하는 예제입니다.

* **로직:** `update()` 함수를 초 단위로 호출하여 현재 시각을 2진수 데이터로 변환합니다.
* **렌더링:** `ReactDOM.render()`가 매초 실행되지만, 리액트의 가상 DOM 덕분에 전체 화면을 새로 고치지 않고 **숫자가 변하는 특정 요소**만 골라 업데이트합니다.

---

## 4. 핵심 포인트 요약

* **중괄호 `{ }` 사용법:** JSX 문법 안에서 자바스크립트 변수나 계산식(표현식)을 넣고 싶을 때는 반드시 중괄호를 사용합니다.
* **효율성:** 가상 DOM은 성능 최적화의 핵심이며, 개발자가 직접 DOM을 조작하는 번거로움을 해결해 줍니다.
* **가독성:** JSX를 통해 HTML과 자바스크립트의 경계를 허물어 더 생산적인 코딩이 가능합니다.
