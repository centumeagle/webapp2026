# 📋 React 기초: 프로젝트 생성 및 MPA vs SPA 완벽 정리

React 입문자를 위한 프로젝트 시작 방법과 핵심 아키텍처 이론 가이드입니다.

---

## 1. React 프로젝트 생성하기 (Vite)

가장 빠르고 가벼운 차세대 빌드 도구인 **Vite**를 사용하는 방법입니다.

### ✅ 실행 순서
터미널(Terminal)에서 아래 명령어를 순서대로 입력하세요.

~~~bash
# 1. 프로젝트 생성
npm create vite@latest my-app -- --template react

# 2. 폴더 이동 및 설치
cd my-app
npm install

# 3. 서버 실행
npm run dev
~~~

## 2. MPA vs SPA 상세 비교

### 🏢 MPA (Multi Page Application)
* **정의:** 여러 개의 HTML 파일로 구성된 전통적인 웹 방식.
* **동작:** 페이지 이동 시 서버에서 새 HTML을 받아오며 **전체 새로고침(깜빡임)**이 발생합니다.
* **장점:** SEO(검색 엔진 최적화)에 유리하며 초기 로딩이 빠릅니다.
* **비유:** 주문할 때마다 새 접시에 처음부터 요리를 다시 담아 내오는 식당.

### 🚀 SPA (Single Page Application) - [React의 방식]
* **정의:** 단 하나의 HTML 파일만 로드하고 자바스크립트로 화면을 갈아끼우는 방식.
* **동작:** 새로고침 없이 필요한 데이터만 업데이트하여 **앱처럼 부드러운 경험**을 제공합니다.
* **장점:** 뛰어난 사용자 경험(UX), 서버 부하 감소, 컴포넌트 재사용 용이.
* **비유:** 큰 접시 하나를 두고, 다 먹은 음식만 새 요리로 계속 갈아주는 방식.

---

## 3. 왜 React는 SPA 방식을 사용할까요?

1. **매끄러운 사용자 경험:** 페이지 전환 시 흐름이 끊기지 않아 빠른 반응성을 제공합니다.
2. **효율적인 데이터 통신:** 무거운 HTML 전체 대신 가벼운 데이터(JSON)만 주고받아 경제적입니다.
3. **컴포넌트 개발 최적화:** 화면을 조각(Component) 단위로 관리하는 React의 철학과 일치합니다.
4. **강력한 생태계:** 라우팅 라이브러리를 통해 SPA에서도 복잡한 페이지 관리가 가능합니다.

## 💡 요약 비교표

| 구분 | MPA (전통적 방식) | SPA (React 방식) |
| :--- | :--- | :--- |
| **페이지 구성** | 여러 개의 HTML 파일 | 단일 HTML 파일 |
| **새로고침** | 이동 시마다 발생 | 발생하지 않음 |
| **속도** | 초기 로딩 빠름 / 전환 느림 | 초기 로딩 느림 / 전환 빠름 |
| **SEO** | 매우 유리함 | 별도 설정 필요 |
| **주요 기술** | Spring MVC, PHP, JSP | **React**, Vue, Angular |
