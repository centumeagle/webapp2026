# 📋 React: 일반 변수 vs State 완벽 가이드

React에서 데이터의 변화를 화면에 즉시 반영하려면 반드시 **State**를 사용해야 합니다.

---

## 1. 일반 변수 vs State 핵심 비교

| 구분 | 일반 변수 (let, const) | State (useState) |
| :--- | :--- | :--- |
| **데이터 변경** | 자유롭게 변경 가능 (`count++`) | `set` 함수를 통해서만 변경 (`setCount`) |
| **리렌더링** | **발생하지 않음** (화면 안 변함) | **발생함** (화면이 즉시 업데이트) |
| **데이터 유지** | 컴포넌트 재실행 시 **초기화됨** | 컴포넌트 재실행 시에도 **값 유지** |

---

## 2. 사용 사례: 언제 무엇을 쓸까?

### ✅ useState를 사용하는 경우 (State)
* **사용자의 입력값:** Input, Checkbox, Form 데이터 등.
* **서버 데이터:** API 호출을 통해 받아온 목록이나 상세 정보.
* **UI 상태 제어:** 모달의 열림/닫힘, 다크모드 토글, 탭 전환 등.
* **결론:** "이 값이 바뀌었을 때 화면(UI)도 같이 바뀌어야 한다"면 무조건 **State**입니다.

### ✅ 일반 변수를 사용하는 경우 (Variable)
* **정적 상수:** 컴포넌트 내부에서 변하지 않는 고정된 값 (예: `const MAX_FREE_TRIAL = 3;`).
* **임시 연산:** 렌더링 도중 계산을 위해 잠깐 쓰고 버리는 중간값.
* **UI와 무관한 로직:** 값이 바뀌어도 화면을 다시 그릴 필요가 전혀 없는 내부 식별값이나 플래그.

---

## 3. 핵심 주의사항

1. **불변성 유지 (직접 수정 금지):**
   * `state = 10;` 처럼 값을 직접 대입하면 React는 데이터가 변한 것을 감지하지 못해 화면을 그리지 않습니다.
   * 반드시 `setState(10)`과 같은 **세터 함수**를 사용해야 합니다.

2. **비동기적 업데이트:**
   * `setState`를 호출한다고 해서 그 다음 줄에서 바로 값이 바뀌는 것이 아닙니다. 
   * React는 성능을 위해 상태 업데이트를 모아서 처리하므로, 변경된 값은 **다음 렌더링 시점**에 반영됩니다.

3. **초기화 현상:**
   * 일반 변수(`let`)는 컴포넌트가 리렌더링될 때마다 코드가 처음부터 다시 실행되면서 **초기값으로 리셋**됩니다. 
   * 반면 `useState`로 만든 값은 리렌더링 사이에도 React가 메모리에 안전하게 보관해 줍니다.
